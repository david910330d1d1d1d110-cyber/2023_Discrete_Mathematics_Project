# 遞移關係的數量求算及演算法優化
> **Discrete Mathematics Project | From Naive to Bit-vector Optimization**

[📄 點此閱讀完整研究報告 (PDF)](<遞移關係的數量求算及演算法優化.pdf>)

本專題探討如何透過程式邏輯，在元素數量為 $n$ 的集合中，計算所有符合「遞移律 (Transitivity)」的關係矩陣總數。我們實作了 **OEIS A006905** 序列的計數，並嘗試透過位元優化克服組合爆炸帶來的運算瓶頸。

---

## 📖 研究動機
在離散數學中，遞移律的定義為：若 $(a, b) \in R$ 且 $(b, c) \in R$，則 $(a, c) \in R$。
雖然定義簡單，但實作計數時面臨極大的效能挑戰：

1. **理論驗證**：產出數據並與 **OEIS A006905** 序列比對，確保演算法在數學上的正確性。
2. **組合爆炸**：關係總數以 $2^{n^2}$ 指數級別成長。當 $n=6$ 時，搜尋空間高達 $2^{36} \approx 6.87 \times 10^{10}$ 種組合。
3. **效能挑戰**：觀察當 $n=6$ 時，高達 **687 億次** 的運算量如何讓普通演算法崩潰，並嘗試優化它。

---

## 🛠️ 演算法實作與優化

本專案從最基礎的檢查邏輯出發，逐步優化至底層位元並行運算。

### 1. 暴力法 (Naive Approach) - $O(n^3)$
使用三層嵌套迴圈檢查所有的三元組 $(i, k, j)$。雖然邏輯直觀，但在面對大數據時效能極低：

```c
for (int i = 0; i < n; i++)
    for (int k = 0; k < n; k++)
        if (Matrix[i][k]) // 若 i -> k 存在
            for (int j = 0; j < n; j++)
                if (Matrix[k][j] && !Matrix[i][j]) 
                    return 0; // 若 k -> j 存在但 i -> j 不存在，則不符合遞移律
```

### 2. 位元並行優化 (Bit-vector Optimization) - $O(n^2)$
將矩陣的每一列存儲為 **Bit-vector**，利用 CPU 位元運算一次處理多個元素的邏輯判斷。
    ```c
    // 關鍵優化：利用位元交集運算快速判斷遞移律
    if ((Row[k] & Row[i]) != Row[k]) return NOT_TRANSITIVE;
    ```

#### 🧪 技術突破與心得
* **Early Exit 策略**：發現第一個反例即中斷檢查，在處理數千萬個矩陣時省下巨大運算成本。
* **底層效能掌控**：在 C 語言中處理**位元運算優先權**與**記憶體映射**，實現了針對大規模數據的效能微調。
* **面對數據的直觀感受**：深刻體會到當 $n$ 增加 1 時，計算量劇烈跳轉的恐怖，理解了演算法優化的價值。
